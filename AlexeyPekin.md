 # Конспект по работе с GIT #

 ## Основные команды в GIT ##
 
 - git init - создаем репозиторий для работы с файлом

 - git add - добавляет собержимое рабочего каталога в индекс для последующего коммита

 - git commit - сохраняет данные и изменения,которые были добавлены с помощью git add 

 - git status - показывает состояние репозотория

 - git log - просмотр всех коммитов,журнал изменений

 - git diff - просмотр изменений до коммита

 - git version - смотрим текущую версию git

  - clear - чистка терминала

  - git config - задаем имя пользователя и почту

  ## Работа с ветками ##

  * git branch - выводим список всех веток
  * git branch new_name - создаем новую ветку new_name
  * git checkout - выбираем нужную нам ветку
  * git merge - слияние двух веток
  * git log --graph - выводим список коммитов в виде дерева/графа
  * git branch -d - удаление ветки

  
   ## При слиянии веток 'с конфликтом' выбираем нужное предложенное нам действие ##

 # примечание 1 # 
 Слияние,когда из вливаемой ветки извлекаются несколько коммитов,а затем они применяются к последнему коммиту целевой ветки,называется fast-forward.

 # примечание 2 #
  При слиянии веток,перед командой merge,убедиться,что мы находимся на целевой ветке,в которую будем вливать изменения из вливаемой ветки.

## Работа с удаленным репозиторием ##

* git clone - копируем себе репозиторий
* git push - отправляем изменения с локального репозитория на удаленный
* git pull - выкачиваем изминения из удаленного репозитория на локальный
 ## fork - делаем копию репозитория с другого аккаунта ##
