# Моя Git Инструкция

> *после установки программы git на свой компьютер, обязательно
авторизуйтесь используя команды:*

## Команды:

### Инициализация репозитория

* *git init* - команда, инициалифлизирующая репозиторий в текущей папке. При инициализации он создаст скрытую папку. В ней содержатся все объекты и ссылки, которые Git использует и создаёт в истории работы над проектом.

### Проверка статуса репозитория

* *git status* - Просмотреть статус нужного репозитория можно по ключевому слову **status**: его действие распространяется на подготовленные, неподготовленные и неотслеживаемые файлы.

### Сохранение изменений и запись коммита

* *git add ./* * - добавить файл к отслеживанию, также для сохранения внесеных изменений

* *git commit -m* - Добавить комментарий к новому сохранению

### log

* *git log* - команда, которая выведет все коммиты
* *git log --graph* - команда, которая выведеть все коммиты и покажет ветки графически.
*  Кроме того, добавив флаг -p, вы можете подробно изучить изменения, внесённые в каждый файл.

### Просмотр изменений до коммита

* *git diff* - Можно просматривать список изменений, внесённых в репозиторий, используя параметр diff. По умолчанию отображаются только изменения, не подготовленные для фиксации.

### Удаление отслеживаемых файлов из текущего рабочего дерева

* *git rm somefile* - Удалять файлы из текущего рабочего дерева можно с помощью параметра rm. При этом файлы удаляются и из индекса.

### Изменение последнего коммита

* *git commit --amend -m "..."* - Внести изменения в последний коммит можно параметром commit с флагом --amend. Например, вы записали изменения, внесённые в ряд файлов, и поняли, что допустили ошибку в сообщении коммита. В этом случае можете воспользоваться указанной командой, чтобы отредактировать сообщение предыдущего коммита, не изменяя его снимок.

### Создание новой ветки и переход в неё

* *git branch branch_name* - Создать новую ветку можно с помощью параметра branch, указав имя ветки.
* После создания ветки переходим в нужную нам ветку с помощью команды **git checkout branch_name** Если после checkout поставить -b то можно переместится на новую ветку создав ее

### Слияние веток

* *git merge branch_name* - команада, которая позволяет слить ветки
* Прервать слияние в случае конфликта можно параметром merge с флагом --abort. Он позволяет остановить процесс слияния и вернуть состояние, с которого этот процесс был начат. *git merge --abort*

### Перемещение по веткам

* *git checkout main* - команда, для возврата к основной ветке
* *git checkout branch_name* - команда, для перемещения по созданым веткам

### Удаление ветки

* *git branch -d name* - далить ветку можно параметром branch с добавлением флага -d и указанием имени ветки. Если вы завершили работу над веткой и объединили её с основной, можно её удалить без потери истории. Однако, если выполнить команду удаления до слияния — в результате появится сообщение об ошибке. Этот защитный механизм предотвращает потерю доступа к файлам.

### Добавление удалённого репозитория
* *git remote add* - Добавить удалённый репозиторий можно параметром remote add, указав shortname и url требуемого репозитория.

### Отправка изменений в удалённый репозиторий
* *git push* - Отправлять изменения в удалённый репозиторий можно параметром push с указанием имени репозитория и ветки.

### Получение изменений из удалённого репозитория
* *git pull* - Для загрузки изменений из удалённого репозитория используется параметр pull. Он скачивает копию текущей ветки с указанного удалённого репозитория и объединяет её с локальной копией.
