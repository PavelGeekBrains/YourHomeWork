# Краткое руководство-конспект по использованию системы Git при помощи визуализатора Visual Studio Code и языка разметки Markdown
## Урок 1 начало работы.

* ## Первым делом нужно создать папку, в которой будет будет существовать репозиторий. Необходимо точно знать по какому пути расположена вышеуказанная папка.
* ## Далее открываем Visual Studio Code и в программе открываем созданную папку.
* ## Следующим шагом нужно открыть терминал (вкладка Терминал, создать терминал) и при помощи команды
 ## *git init* 
  ## произвести инициализацию репозитория в текущей папке. В папке будет создан файл .git

* ## Репозиторий готов к работе.

## Теперь нужно создать новый файл, или открыть существующий  для дальнейшего редактирования.
* ## Создаем новый файл любым удобным способом (из меню Файл, сочетанием клавиш, и т.д.). Если нужно чтобы работа происходила в среде языка разметки Markdown, необходимо указать для файла расширение .md
## Cозданный файл должен изначально быть окрашен зеленым цветом и иметь справа метку U, свидетельствующую о том, что на данный момент файл пока не отслеживается.
* ## Далее происходит работа с файлом, до того момента, как возникнет необходимость создать сохранение.
* ## Для сохранения первым делом нужно применить сочетание кклавиш *Ctrl + S*, после чего белая точка справа от вкладки файла исчезнет.
* ## Следующим действием будет добавление изменений в файл репозитория. Для этого сучествует команда
## *git add*
## также после введения команды, через пробел при помощи нажатия клавиши Tab, выбираем текущий файл и нажимаем Enter.
## Если изменения произошли без ошибок, то система перейдет на следующий шаг, а справа от названия файла буква U сменится на букву А, это означает что изменения успешно добавлены.
* ## Далее нужно зафиксировать (сохранить) изменения. Для этого потребуется команда
## *git commit*
## также после самой команды необходимо приложить к коммиту описание. Для этого вводим ключ -m"***" где внутри кавычек будет текстовое описание сохранения.
## по выполнению *git commit* система сообщит о том, что сохранения произошло успешно, и сколько произошло изменений. Изменения желательно фиксировать как можно чаще, чтобы не потерять их безвозвратно.
* ## Для просмотра предыдущих версий файла нужна команда
## *git log*
## Этой командой выводятся все коммиты в хронологическом порядке с указанием времени изменения файла и уникальным идентификатором. Для перемещения между коммитами , если внизу виден знак двоеточие, удобно воспользоваться стрелками "вверх/вниз". Для возврата к терминалу нужно нажать клавишу Q.
* ## Для перехода к нужному коммиту, созданному ранее, потребуется команда
## *git checkout*
## сочетание *git checkout* и первых четырех знаков уникального идентификатора приведут к переходу в нужную версию докумета, которая была сохранена ранее.
* ## Для возврата к текущей версии документа применяется команда
## *git checkout master*
## Выполнение этой команды вернет пользователя в последнюю текущую версию документа.
* ## Для промежуточного просмотра статуса изменений существует команда
## *git status*
## Команда отображает текущий каталог и расположение измененных файлов. С ее помощью можно проверить внесенные изменения.
* ## Cинтаксис языка разметки Markdown широко представлен в сети интернет.

## Урок 2 ветвление

* ## Для того, чтобы сохранить основной документ нетронутым, и в то же время вносить черновые изменения или усовершенствовать документ усилиями нескольких человек, применяется ветвление. Для создания альтернативной ветки используется команда
## *git branch new_branch_name*
## где new_branch_name название вновь создаваемой ветки.
* ## для просмотра существующих веток и опеделене текущей вветки используется команда 
## *git branch*
## после выполнения команды звездочкой будет помечена текучая из существующих веток.
* ## Для переключения между ветками используется команда
## *git checkout branch_name*
## где branch_name название ветки назначения.
* ## По окончании внесения изменений в различных ветках документа, администратором, ответственным за целостность проекта производится слияние веток. Слияние выполняется командой
## *git merge name_branch*
## где name_branch имя ветки донора. Слияние произойдет в текущую ветку.
## Существует несколько сценариев развития событий при слиянии.
1. ## Были внесены правки только в новую ветку, и она хронологически опередила ветку Master. Такое слияние называют *Fast Forward*. Фактически при этом происходит хронологическое выравнивание двух веток.
2. ## Внесенные правки в обе ветки не пересекаются и не влияют друг на друга. В этом случае слияние так же произойдет без конфликта
3. ## Внесенные правки пересекются с данными из разных веток, и в этом случае возникает конфликт, который система поручает разрешить пользователю. Пользователю предлагается на выбор:
## - принять текущие изменения
## - принять входящие изменения
## - принять оба изменения
## - сравнить изменения с оригиналом

4. ## Слияние происходит с игнорированием части данных.
## Например добавим фото из папки репозитория:
![image 1](IMG_20200620_165618.jpg) 
## Поскольку в Git не принято загружать большие файлы, файл нужно проигнорировать. Для этого в папке репозитория создается файл *.gitignore* и в него записывается имя файла изображения. Добавляется в .gitignore и создается коммит. После этого можно выполнить слияние.
* ## Ко всему прочему, при помощи команды
## *git log --graph
## можно вывести на экран лог с трек-линиями создания и слияния веток.
* ## По завершению всех слияний, слитые ветки можно удалить при помощи команды
## *git branch branch_name*
## где branch_name имя ненужной уже слитой папки.

## Урок 3 удаленный репозиторий

* ## Для создания удаленного репозитория на Github, после регистрации на сайте нужно зайти во вкладку Your Repositories, нажать зеленую кнопку NEW, задать имя для репозитория, и нажать Create Repository.
* ## Чтобы зоздать клон репозитория на локальном компьютере, используется команда 
## *git clone* + ссылка на созданный удаленный репозиторий
## Ссылка находится под зеленой кнопкой Code внутри репозитория. Клон репозитория помещается в папку на локальном компьютере. Далее пользуясь подсказками в Github можно в командной строке создать новый репозиторий или отправить существующий. 
* ## Для отправки файлов в удаленный репозиторий используется команда
## *push*
* ## Для получения файлов из удаленного репозитория и синхронизации данных загруженных с другого компьютера, а также измененных в самом удаленном репозитории, используется команда 
## *pull*
* ## Можно создать на Github копию удаленного репозитория, принадлежащего другому человеку и поместить её в расположение своих репозиториев при помощи кнопки Fork. Затем в Visual Studio Code последовательностью действий: *Проводник-Копировать репозиторий-Клонировать из GitHub* создаем клон "форкнутого" репозитория и помещаем его в папку на локальном компьютере. После этого можно производить редактирование файлов клонированного репозитория, затем создать коммит и отправить его обратно в удаленный репозиторий командой *git push*
* ## Чтобы отправить отредактированные данные владельцу репозитория, нужно пройти последовательность действий: *Pull reqests - New pull request - Create puii request*, после чего владелец репозитория получит запрос на слияние отредактированной информации с оригиналом.
