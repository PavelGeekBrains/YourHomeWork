# My git instruction

## Btginning of work:
1. Создайте папку в которой будет находиться ваш репозиторий.
2. Через проводник VSC откройте созданную папку.
3. Откройте терминал VSC.
4. Убедитесь в том, что система контроля версий GIT установлена и корректно работает, используйте для этого команду "git version" в терминале VSC.
5. Инициализируйте создаваемый вами репозиторий, используйте для этого команду "git init" в терминале VSC.
6. Создайте файл для работы в репозиторие.
7. Добавьте созданный файл в систему контроля версий GIT, используйте для этого команду "git add имя файла с расширением", при этом статус файла изменится с неотслеживаемого на отслеживаемый системой GIT.
8. Зафиксируйте состояние, в последующем изменения созданного файла, используйте команду "git commit -m "комментарий к изменению файла"".
9. Начните работу.
10. Информацию по работе с GIT Вы можете найти в интернете.

# Работа с удаленным репозиторием
Репозиторий создается на портале GITHUB. У одного локального репозитория может быть связь только с одним удаленным репозиторием.

# Команды используемые для работы
* *git add* - сохраняет снимок текущего состояния проекта;
* *git commit* - записывает индексированные изменения в репозиторий;
* *git log* - выводит историю репозитория;
* *git log graph* - выводит коммиты в виде дерева;
* *git status* - отображает состояние рабочего каталога, файла;
* *git checkout* - позволяет перейти к какой либо версии файла;
* *git diff* - выводит изменения между вашей текущей версией файла и сохраненной;
* *git init* - позволяет начать отслеживание изменений в проекте;
* *git branch* - команда, отображающая список существующих веток;
* *git branch -d branch_name* - удаляет ветку "branch_name";
* *git merge branch_name* - команда, осуществляющая слияние текущей ветки с branch_name;
* *git --help* - выводит общуюдокументацию по GIT;
* *git log --help* - выводит документацию по конкретной команде;
* *git pull* - отвечает за синхронизацию удаленного содержимого;
* *git push* - отправляет локальную ветку на удаленный репозиторий;
* *git fetch* - загружает коммиты, файлы и ссылки из удаленного репозитория в локальный;
* *git branch -M master* - делает главной ветку master в удаленном репозитории;
* *.gitignor* - позволяет игнорировать все файлы прописанные внутри;
* *git remote add origin https* - позволяет добавить базовый репозиторий по какой либо ссылке;
