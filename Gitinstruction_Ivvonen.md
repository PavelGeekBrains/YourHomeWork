# Инструкция по работе с Git

## Как начать работу

* *__git --version__* - команда, позволяющая узнать версию установленной программы Git.

* *__git init__* - команда, инициализирующая репозиторий.

* *__git status__* - показывает текущее состояние git, есть ли изменения, которые нужно сохранить (закоммитить).

## Как сохраниться

* *__ctrl+s__* - чтобы сохранить изменения.

* *__git add__* - добавляет собержимое рабочего каталога в индекс для последующего коммита. 

* *__git commit -m"text"__* - зафиксировать, сохранить. Команда git commit берёт все данные, добавленные в индекс с помощью git add, и сохраняет их
слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок.

## Как работать с сохранениями

* *__git log__* - журнал изменений. Показывает количество сохранений.

* *__git checkout__* - позволяет переходить от одного коммита к другому, а также менять ветки.

* *__git checkout master/main__* - позволяет вернуться на главную ветку.

* *__git diff__* - показывает разницу между текущим файлом и сохраненным.

## Создание веток

Чтобы создать новую ветку, нужно ввести:

* *__git branch__* <название новой ветки>. 

## Работа с ветками

* _**git checkout <имя ветки>**_ - позволяет переходить в нужную ветку.

* __*git checkout -b [имя ветки]*__ - чтобы создать ветку и сразу же перейти в неё.

* __*git merge <имя ветки>*__ - слияние ветки с текущей.

* __*git rebase [branch name]*__ - ещё один способ объединения изменений в ветках. При этом git копирует набор коммитов и переносит их в другое место.

* *__git branch -d <имя ветки>__* - команда для удаления ветки.

* *__git log --graph__* - отображает графически коммиты в виде дерева.

## Относительные ссылки

* Чтобы переместиться на 1 коммит назад, можно использовать ^ и команду checkout. Например:

*git checkout main^* - перенесет на 1 коммит назад.

* Чтобы переместиться на несколько коммитов назад, можно использовать ~<num>. Например: 

*git checkout main~2* - перенесет на 2 коммита назад.

* Чтобы прикрепить ветку к коммиту, используется  команда:

__*git branch -f main HEAD~3*__ - переместит ветку main на 3 родителя назад от HEAD.

## Отмена изменений в Git

* __*git reset*__ - отменяет изменения, перенося ссылку на ветку назад, на более старый коммит. например, *git reset HEAD~1* перенесет на 1 коммит назад от HEAD.

* __*git revert*__ - перенесёт ветку назад, как будто некоторых коммитов вовсе и не было.

Для сетевых репозиториев подходит revert. Reset хорошо работает для локальных репозиториев.

## Работа с удаленным репозиторием

Копировать внешний репозиторий на свой ПК можно командой **git clone**.

Команда git clone составная: она не только
загружает все изменения, но и пытается слить все ветки на локальном компьютере и в
удаленном репозитории.

__*git pull*__ позволяет скачать все из текущего репозитория и автоматически
слить с нашей версией.

__*git push*__ позволяет отправить свою версию репозитория во внешний репозиторий. **ТРЕБУЕТ АВТОРИЗАЦИИ** на внешнем репозитории при первом её использовании.

## Как настроить совместную работу локального и удалённого репозиториев

1. Создать аккаунт на GitHub.com
2. Создать локальный репозиторий
3. “Подружить” ваш локальный и удалённый репозитории, используя инструкции GitHub. 
4. Отправить (push) ваш локальный репозиторий в удалённый (на GitHub), при этом, авторизоваться на удалённом репозитории.
5. Провести изменения с удаленного репозитория или с другого компьютера.
6. Выкачать (pull) актуальное состояние из удалённого репозитория.

## __pull request__ 

Это запрос на вливание изменений в репозиторий. 

Предлагать изменения на GitHub нужно в отдельной ветке. Сначала
пользователь копирует репозиторий на свой компьютер, делает fork репозитория, затем
клонирует версию на своём ПК, создаёт ветку с предлагаемыми изменениями, отправляет
изменения командой push в свой аккаунт на GitHub и даёт команду pull request. 

## Как сделать pull request

1. Делаем   (ответвление) репозитория fork.

2. Делаем *git clone* **СВОЕЙ** версии репозитория. 

3. Создаем новую ветку и в **НЕЕ** вносим свои изменения

4. Фиксируем изменения (делаем коммиты).

5. Отправляем свою версию в свой GitHub.

6. На сайте GitHub нажимаем кнопку pull request.
