# Руководство по работе с Git'ом


## Базовые команды git

* **git init** - создать новый проект в текущей директории;
* **git status** - показать состояние репозитория (отслеживаемые, изменённые, новые файлы и пр.);
* **git log** - команда, отображающая логи коммитов;
* **git add *file_name*** - добавить в индекс все новые, изменённые, удалённые файлы из текущей директории и её поддиректорий;
* **git checkout** - отменить изменения в файле, вернуть состояние файла, имеющееся в индексе;
* **git commit -m "Name of commit"** - зафиксировать в коммите проиндексированные изменения (закоммитить), добавить сообщение;
* **git checkout b9533bb** - переключиться на коммит с указанным хешем (переместить HEAD на указанный коммит, рабочую директорию вернуть к состоянию, на момент этого коммита);
* **git checkout master** - переключиться на коммит, на который указывает master (переместить HEAD на коммит, на который указывает master, рабочую директорию вернуть к состоянию на момент этого коммита);

## Особенности работы с репозиториями 

### Cоздание директория проекта, мы в ней, указана последовательность действий:

1. **git init**                      - создаём репозиторий в этой директории;
2. **touch readme.md**               - создаем файл readme.md;
3. **git add readme.md** - добавляем файл в индекс;
4. **git commit -m "Старт"**         - создаем коммит;
5. **git remote add origin https://______.com:nicothin/test.git** - добавляем предварительно созданный пустой удаленный репозиторий;
6. **git push -u origin master**     - отправляем данные из локального репозитория в удаленный (в ветку master);

## Перемещение между коммитами

### Для перемещения между коммитами необходимо использование следующих команд:

1. Вызвать команду **-git log**;
2. Выйти из *log* используя клавишу **q**;
3. Узнать уникальный код(достаточно первые 4-6 цифр) нужного коммита, например 9cv45b;
3. С помощью команды __git checkout ****__ (где вместо **** указать первые символы нужного коммита) вызвать нужный нам коммит.
4. Для возврата неободимо вызвать команду **git checkout master**


## Работа с ветками

* **git merge** - команда осуществляющая объединение нескольких веток в одну;

* **git branch** - команда отображающая существующие ветки в репозитории;

* **git branch *branch_name*** - команда создающая новую ветку в репозитории;

* **git log --graph** - команда показывающая ветки в графическом представлении

## Работа с локальным и удаленным репозиториями

* **git pull** - "тянет" данные из удаленного репозитория на локальный 
* **git push** - "толкает" данные из локального репозитория в удаленный
