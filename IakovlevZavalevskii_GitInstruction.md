
# Git instructions. 

### 1. Начало работы

* для инициализации репозитория используется команада *git init*.

### 2. Работа в репозитории

* для добавления файла для отслеживания используется команда *git add*.

* для фиксации изменений файла используется команда *git commit*.

* для вывода журнала изменений файла используется команда *git log*, а для графического и наглядного вывода - _git log -graph_.

* для ввызова конкретной версии файла используется команда *git checkout*.

### 3. Создание новых веток и переход между ними

 * для просмотра списка существующих веток используется команда *git branch*
 
 * для создания новой ветки используется команда *git branch branch_Name*, где _branch_Name_ - название создаваемой ветки

 * для переключения с одной ветки на другую используется команда _checkout branch_Name_ или *swift branch_Name*, где _branch_Name_ - это наименование ветки, на которую идет переключение

### 4. Слияние веток

* для слияния веток необходимо находиться на той ветке, куда будет происходить слияние.

* для осуществления слияния веток используется команда _git branch branch_Name_, где _branch_Name_ -  название ветки, которую сливают с текущей

    
### 5. Конфликты при слиянии веток

* конфликты при слиянии веток возникают тогда, когда коммиты сливаемых веток содержат противоречия.

* для устранения конфликтов необходимо выбрать предлагаемые варианты.

* терминал может сам решить конфликты, если нет явных  противоречий.    

### 6. Работа с удаленными репозиториями

* для работы с удаленным репозиторием необходимо использовать специальный сайт - *github.com*.

* для создания удаленного репозитория необходимо в разделе *Repositories* в личном кабинете нажать нажать на *New*, затем следовать инструкциям.

* перед загрузкой репозитория на *github.com* необходимо сперва последовательно в терминале ввести команды:

    1. git remote add origin https://github.com/GitIT1979/Repository_Name.git, где "Repository_Name" - наимнование созданного удаленного репозитория;

    2. git branch -M main;

    3. git push -u origin main.

* для копирования удаленного репозитория используется команда *git clone repository_Address*, где _repository_Address_ - Интернет-адрес такого репозитория.

* для выгрузки локального репозитория в удаленный используется команда *git push*.

* для загрузки удаленного репозитория в локальный используется команда *git pull*.

* для того, чтобы предложить изменения в чужой удаленный репозиторий необходимо:

    1. зайти в чужой удаленный репозиторий, нажать на *Fork*, опционально дать назавание и описание такому репозиторию, и далее *Create fork*;
    2. скопировать его;
    3. в отдельной ветке внести соответствующие изменения;
    4. создать в репозитории файл *READ_ME*, где описать предлагаемые изменения;
    5. выгрузить локальный репозиторий в удаленный;
    6. в удаленном депозитарии нажать на *Compare and pull request*, опционально дать описание и пояснения, затем нажать на *Create pull requst*.
