# **Инструкция по работе с Git**

## __Базовые команды__

* *``git init``* - команда, инициализируюшая репозиторий;

* *``git commit``* - команда, которая фиксирует добавленные в индекс изменения;

* *``git add "``file name``"``* - команда, которая добавляет содержимое рабочего файла в индекс для последующего коммита;

* *``git status``* - команда, которая отображает список измененных, добавленных и удаленных файлов;

* *``git log``* - команда, показывающая историю произведенных команд;

* *``git commit -m"``message text``"``* - команда, с помощью которой можно создавать комментарии к коммитам;

* *``git branch``* - команда, которая показывает текущую ветку;

* *``git branch branch_name``* - команда, создающая ветку с именем *branch_name*;

* *``git log``* - команда, позволяющая посмотреть историю коммитов;

* *``git checkout``* - команда, позволяющая переключаться между ветками;

* *``git diff``* - команда, которая позволяет увидеть изменения по сравнению с последним коммитом;

* *``git log --graph``* - команда, которая позволяет отобразить созданные коммиты в виде графа/дерева;

* *``git branch -d branch_name``* - команда, удаляющая ветку branch_name.

## __Команды для работы с внешним репозиторием__

* *``git clone``* - команда, которая позволяет склонировать внешний репозиторий на наш ПК;

* *``git pull``* - команда, которая позволяет скачать все из текушего репозитория и автоматически сделать merge c нашей версией;

* *``git push``* -команда, которая позволяет отправить нашу версию репозитория на внешний репозиторий. __ТРЕБУЕТ АВТОРИЗАЦИИ__ на внешнем репозитории.

## __Как сделать pull request__

1. Делаем fork репозитория.
2. Делаем clone своей версии репозитория.
3. Создаем новую ветку и в нее вносим свои изменения.
4. Фиксируем изменения (делаем коммиты).
5. Отправляем свою версию в свой аккаунт GitHub.
6. На сайте GitHub нажимаем кнопку pull request.
