## Git Tutorial

## First Seminar

* git version - версия установленой GIT

* git init -  инициализация гит в репозитории

* git add имя файла(TAB поможет) 

* git commit создание точки сохранения изменений и внесейний правок.

* git add - указание на файл в репозитории для отслеживания

* git log - лог комммитов их дат, номеров, авторов итд.

* git add * - отслеживание всех файлов в репозитории одной командой.

## Hot Keys

* Вызов терминала ctrl + `

* Вызов окна файла ctrl + caps

* Сохранение изменений в файле сtrl + S

## SecondSeminar 

* Fast Forward - update branch master(мастер без commit, ветку которую будем мерджить в мастер с комитом и редакцией но без конфликта)

* ORT *при конфликте* и *без конфликта* :

1. При конфликте - нужно выбрать либо входящие данные, либо исходные или оставить все, обе ветви самостоятельные.

2. Без конфликта - Auto-merging и автоматичски ставит комит на ветке мастер при слиянии(если сливают из мастера) 

* git branch - все ветви

* git branch branch_name - создание ветви
* Просто что то о комитах)

## ThirdSeminar 

* git remote and add origin _link_ (Origin - название удаленного репозитория, link ссылка на него)

* git branch -M main ( main - название ветки) - указание какая ветка основная

* git push -u origin main (отправка в удаленный репозиторий)

* git pull - выгрузка из гитхаба и в то же время merge 

* git clone _link_ на репозиторий который я хочу себе выгрузить
