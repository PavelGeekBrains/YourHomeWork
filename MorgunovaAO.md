# Инструкция по работе с Git

## Базовые команды с первого семинара

* *git init* - команда, инициализирующая репозиторий;
* *git status* - показывает текущее состояние гита, есть ли изменения, которые нужно закоммитить (сохранить);
* *git add* - добавляет самую актуальную версию файла с последними изменениями для последующего коммита. После **git add** с помощью кнопки TAB можно выбрать название нужного файла;
* *git commit* - фиксация (сохранение) изменений. После **git commit** следует ввести сообщение (комментарий) в виде -m "Сообщение, отражающее суть внесенных изменений". Сокращение "m" от англ. "message" - сообщение;
* *git log* - журнал изменений. Используется для просмотра истории коммитов, начиная с самого свежего к первому.
* *git diff* - позволяет сравнивать два различных коммита. Сначала нужно определить хеш (ID) коммитов, которые требуется сравнивать. Это можно сделать с помощью git log. После **git diff** указываем первые 4 символа хеша нужного коммита;
* *git checkout* - переключение между версиями или ветками. После **git checkout** следует ввести первые 4 символа хеша (ID) коммита или название нужной ветки;
* *git --version* - показывает актуальную версию программы Git. Проверяем, видит ли наш редактор исходного кода систему контроля версий Git;
* *clear* - очищение терминала.

## Базовые команды со второго семинара

* *git branch* - команда, отображающая в терминале список существующих веток. Текущая ветка будет отмечена звездочкой (*);

* *git branch branch_name* - команда, создающая ветку с именем *branch_name*;

* *git checkout branch_name* - команда, позволяющая переключаться между ветками; 

* *git merge branch_name* - команда для слияния текущей ветки с другой веткой (ее название мы указываем вместо branch_name);

### Ситуации, возникающие при слиянии веток: 

**Fast-forward** (“перемотка вперед”) - режим слияния веток, при котором все изменения происходили только в новой ветке, в master не было внесено никаких изменений. Последний коммит, сделанный во входящей ветке, после сляения будет отображен как последний коммит в текущей ветке (master). Противоречий нет, конфликт решился автоматически.  

**Ort strategy** - автоматическое слияние без противоречий, изменения были внесены в обоих ветках, но в разных местах. Автоматически создается коммит о слиянии веток. 

**Конфликты** - ситуации, при которых Пользователю предлагается разрешить обнаруженные противоречия. Название файла в Проводнике выделено красным цветом, слева - восклицательный знак. Предлагаемые варианты: 
1. Current change - оставить изменения из текущей ветки;

2. Incoming change - оставить изменения из входящей ветки;

3. Accept both changes - оставить оба варианта изменений;

4. Compare changes - сравнить изменения.

Конфликт считается разрешенным после создания коммита о его разрешении. 

* *git log --graph* - визуализация в терминале всех веток и изменений в них в виде графа. Самая левая ветка - master;

* *git -d branch_name* - удаление ветки с именем branch_name.

## Базовые команды с третьего семинара

* *git clone* - позволяет склонировать удаленный репозиторий на наш ПК. Команда git clone составная: она не только загружает все изменения, но и пытается слить все ветки на локальном компьютере и в удаленном репозитории. После git clone необходимо указать ссылку на удаленный репозиторий.
