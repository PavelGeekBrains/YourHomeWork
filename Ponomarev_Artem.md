# Руководство по работе с GITом

## Базовые команды git

git --version версия программы

git config --global user.name "имя автора" - задать имя пользователя

*git config --global user.email"адрес почты"*

*git init* - инициализация папки, в которой работаем;

*git add* назв.файла.расширение

*git log* - список всех коммитов
*Git add* - добавляет актуальную версию

*cd завание папки* переход между папками внутри папки

## Базовые команды с 2 семинара

*git branch* - команда показывающая все сущ. ветки в репозитории
*git branch name* - создание новой ветки в репозитории
*git log --graph* - графическое отображение коммитов
*git marge name* - слияние веток (перенес с другой ветки в текущую)
*git branch -d name" - удаление ветки (d лучше маленькую")


## Особенности работы с репозиториями

U около названия папки значит файл есть но не отслежен

A около названия значит отслеживается 

M значит файл изменен

Белая точка рядом с названием файла, значит не сохранен



*git commit -m "Комментарий"* добавление комментария об изменениях

*git diff* - отличие сохраненной версии от не сохраненной

*git log* - журнал изменений


## Перемещение между коммитами
*git checkout master* - венуться к рабочей версии

*git checkout* -переход к списку версий

Нажать Q при большом списке изменений, если выйдет end

## Варианты при конфликте сливания веток
*Current change* - оставляет из ветки мастер( в которую добавляем)
*incomming change* - оставляет из ветки, из которой добавляем
*both change* - оставляем оба варианта

## Работа с удаленным репозиторием GitHub

*git clone "ссылка с github"* - копирование на локальный репозиторий с сетевого

* git remote add "origin" "ссылка"* - привязка к сетевому репозиторию

*git push -u origin "назв.ветки"* - выбор ветки для отправки в сетевой репозиторий. После можно просто *git push*

*git pull* -отправить в сетевой репозиторий

Процесс работы с удаленным репозиторием:
1.Подружить сетевой с локальным
2.Отправить push
3.Скачать pull
4.Клонировать
