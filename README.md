## __Basic commands and procedure__ 

* *git init* - команда, инициализирующая репозиторий в текущей папке

* *git status* - команда для отображения текущего состояния репозитория

* *git log* - история коммитов

## __Добавление в репозиторий и сохренение__

* *git add* - команда, добавляющая в индекс все новые, измененные, удаленные файлы из текущей директории
* *git commit -m "..."* - команда для фиксации добавленных в индекс фалов с комментариями к ним
* *git commit --amend -m"..."* - команда для изменения последнего коммита, с записью нового комментария"

## __Ветки__

* *git branch* - команда для вывода имеющихся веток в репозитории;

* *git branch branch_name* - команда для создания новой ветки;

* *git checkout branch_name* - команда для переключения с текущей ветки на ветку branch_name;

* *git merge branch_name* - команда, осуществляющая слияния текущей ветки с branch_name;

* *git merge --abort* - команда, прерывающая слияние в случае конфликта, он  позволяет остановить процесс слияния и вернуть состояние, с которого этот процесс был начат.
 
## _Conventional symbols_

+ U - неотслеживаемые изменения

+ M - измененный файл

+ ! - конфликт

## Работа с удаленными репозиториями

* оформляем новый репозиторий на github

* создаем на компьютере пустую папку - можно с таким же названием, как и на github

* при помощи VSD открываем созданную папку

* в созданной папке создаем файл README.md

* инициализируем новый репозиторий и фиксируем изменения (git init, git add, git commit)

* git branch -M master - команда для установки базовой ветки

* git remote add origin ...(adress) - добавляем удаленный репозиторий по указанной ссылке

* git push -u origin master - команда, которая направляет текущую версию локального репозитория на удаленный. -u - модификатор, обозначающий, что даная ветка имеет статус upstream (доступна к отправке на удаленный репозиторий)

* У одного локального репозитория может быть связь только с одним удаленным 

* git push - команда для отправки в удаленный репозиторий внесенных изменений (git add, git commit), сделанных в локальном репозитории (команда идет без дополнительной информации, поскольку ранее оба репозитория уже были соединены)

* При внесении изменений на github и создания commit, для обновлении информации в локальном репозитарии используется команда - git pull

* fork - данная плашка на стороннем открытом репозитории в github позволяет создать копию данного репозитория у себя в github

* clone git repository in VSD - далее в верхнем окошке нажимаем clone from github - подтвердаем свою идентификацию - затем выбираем нужный репозиторий - создаем новую папку и выбираем ее для расположения выбранного репозитория

* git checkout -b branch_name - создаем отдельную ветку в клонированном репозитории

* Далее вносятся изменения в файл и фиксируются (git add, git commit)

* git push --set upstream origin branch_name - данная команда отправляет и одновременно создает в удаленном репозитории ветку с именем branch_name (также можно вызвать команду git push - и система сама подскажет верную команду)

* После этого на странице в github в удаленном склонированом репозитории появляются данные о поялении новой ветки и плашка "Compare & pull reqest"

* Нажав на данную плашку, есть возможность убедиться с какой ветки и на какую предполагется слияние, а также обратиться к владельцу базовой ветки с пояснением о предлагаемом слиянии
