  # **Семинар 1**
  * ***git config --global user.name*** - команда для ввода имени;
  * ***git config --global user.email myEmail@example.com*** - команда для ввода почты; 
  
    ``Первые две команды,нужны для того чтобы наши действия были отмечены именем и почтой. Это помогает понять, кто отвечает за какие изменения ``
  * ***git init*** - создание репрозиторий (подключение к нашей папке и создание скрытой папки .git);
 
     ``В  будет папке, котору вы создадите,будет храниться история репрозитория и его настройки``
  * ***git status*** - команда, показывающая текущее состояние репрозитория: актуально ли, что поменялось и т.д. ;
 
  * ***git add*** - добавить файл;
 
     ``С помощью этой команды, вы можете добавить файлы разных расширений ``
  * ***git commit -m "Add some text"*** - сохранить изменения;
 
     ``Коммиты позволяют нам зафиксировать изменения в файлах. Добавляем файлы командой git add, после создаем сам коммит после -m в кавычках описываете изменения (комментарий) ``
  * ***git log*** - просмотр сохраненных коммитов;
 
     ``Для просмотра всех зафиксированных коммитов можно воспольлзоваться этой командой. Здесь указанны сведения о каждом коммите ``
  * ***git checkout*** - переключение между версиями;
 
     ``Благодаря команде checkout, позволяет переключаться между коммитами (можно выбрать разный коммит, подставляя номер коммита) ``
  * ***git checkout master*** - возращение на актуальное состояние;
  * ***git diff*** - разница между текущим файлом и сохраненным;
