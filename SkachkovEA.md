**# Шпаргалка по git`у**

**## Команда** **git init**

Создаёт в текущем каталоге новый подкаталог с именем .git.

**## Команда** **git add******

Добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита.

**## Команда** **git status**

Отображает состояние рабочего каталога и раздела проиндексированных файлов.

**# __Git Урок №2__**

**## Работа с ветками**

Команда **git status** позволяет просмотреть текущее состояние, в том числе - название текущей ветки.

**### Добавление новой ветки**

Для создания новой ветки необходимо ввести команду:

git branch имя_ветки

**### Переход между ветками**

Для перехода между ветками необходимо ввести команду:

git checkout имя_ветки

**#####** **Проверка выбора ветки git branch**

**### Удаление ветки**

Для удаления ветки необходимо ввести команду:

git branch -d имя_ветки

- *Внимание!** - перед удалением ветки ПРОВЕРЬ, что данные сохранены (ветку слили) и данная ветка более не потребуется!

**### Слияние веток (безконфликтный и конфликтный)**

Для слияния (двух) веток применяется следующая команда:

git merge имя_ветки

- ***ВАЖНЫЕ МОМЕНТЫ при слиянии:****
- В момент слияния мы находимся в ветке, куда будет загружены изменения;
- Имя_ветки выбирай ту, изменения которой будут отправляться!

**### Последствия слияния веток**

При слиянии могут возникнуть ситуации, когда в основной ветке есть противоречия отностительно изменений,  получаемых с побочной ветки. В таких случаях в среде VS Code нам предоставляется возможность реализовать один из следующих вариантов :

- 1. Оставить в основной ветке её версию данных;
- 2. Заменить данные в основной ветке на данные вливаемой ветки;
- 3. Оставить оба варианта, для возможности вручную указать итоговые данные.


# Cеминар3
## Работа с удаленными репозиториями.
1. Создание/авторизация на GitHub
2. Создание удаленного репозитория через страницу своего аккаунта/переход на репозиторий необходимый с помощью полученной ссылки.
3. После перехода на созданный репозиторий включаем "Fork" для создания копии данного репозитория в нашем аккаунте.
4. Для работы с репозиторием необходимо скопировать на локальный компьютер. через кнопку "Code"
5. Работа с полученным репозиторием в основном происходит на новой ветке.
6. По выполнению задачи для которой создана ветка, отправляем на  GitHub данную ветку. 

  git push origin branch_name

7. Переходим на GitHub и нажимаем "Pull requests" где можно указать сообщение для владельца репозитория.

Команды работы с удаленным репозиторием.

 git clone
 git pull
 git push (git push origin branch_name)
