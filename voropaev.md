# **Инструкция по работе с GIT**

*1. Введение*

Git - система контроля (управления) версиями. 
Создатель - Линус Торвальдс.
Git необходима для отслеживания и ведения истории изменения данных в файлах, также при командной работе над файлами.

*2. Команды Git*
* **git init** - команда для инициализации репозитория.  Необходима чтобы Git начал отслеживать папку (репозиторий).

* **git add** - команда, необходимая для того, чтобы Git сохранил текущую версию (снимок) файла, для последующего ввода комментария (коммита). После команды необходимо ввести наименование файла.

* **git commit -m** - команда для добавления комментария текущей сохраненной (добавленной) версии (снимка) файла. После команды необходимо ввести комментарий в ковычках.

* **git status** - команда, отображающая состояние файлов в текущей папке. Показывает имеются ли изменения в файле, требующие сохранения и коммита или изменения в файле относительно последней сохраненной версии отсутствуют.

* **git log** - команда, отображающая сохраненные версии (снимки) файла с комментариями. **Q** - выход из режима отображения версий файла в окне терминала.

* **git checkout** - команда, позволяющая перейти к любой версии сохраненного файла. Для этого после команды необходимо ввести полное наименование или первые четыре символа наименования комита. Пример *git checkout fa1c7b054e96ef132fe71594e1fa7d88d1205d48* или *get checkout fa1c*.

* **git checkout master** - команда, позволяющая перейти к последней актуальной версии файла.

* **git clear** - команда, позволяющая очистить окно терминала.

____

Напоминание:

Для сохранения текущей версии файла в программе Git необходимо:
1. В окне редактирования текста выполнить команду Ctrl + S;
2. Выполнить команду git add;
3. Выполнить команду git commit -m.

# Команды Git со второго семинара.

* *git branch* - команда, которая отображает все доступные ветки, текущая ветка указана символом звездочкой (*) и ее наименование выделено зеленым цветом;

* *git branch branch_name* - команда для создания ветки;

* *git checkout branch_name* - команда для перехода на ветку branch_name;

* *git **merge** branch_name* - команда для добавления информации из ветки branch_name. Для этого (**до этого**) необходимо перейти в ту ветку, в которую нужно вставить информацию.

* *git branch -d branch_name* - удаление ветки branch_name;

* *git log --graph* - вывод список всех комитов в виде дерефа(графика);

* *Чтобы при отображении статуса файлов (командой git status) git игнорировал какие-либо файлы, необходимо создать в рабочем каталоге файл .gitignore и в нем прописать название файла с расширеннием, которое необходимо gitу проигнорировать* - добавлена также в мастер правильная информация посмотрим он даст возможность поэтапно выбрать что заменить что оставить

# Команды Git с третьего семинара.

# **GitHub** - сервис от Microsoft по удаленной работе с программой Git.

## Для того, чтобы создать новый репозиторий на GitHub:

(Так как основа для удаленного репозитория - локальный репозиторий, первоначально создаем локальный репозиторий на ПК в программе Git)

1. Создаем папку, инициализируемся в ней:
- **git init**;

2. Создаем рабочий файл, сохраняем его, коммитим:
- **git add;
- **git commit -m;

3. Говорим какая ветка является главной (например **master** или **main**):
- **git branch -M "branch name"**

4. Добавляем ссылку на удаленный репозиторий (Предварительно на GitHub создем новый репозиторий и копируем адрес пустого-созданного-удаленного репозитория): 
- **git remote add origin https://github.com/voropaev56/18.06-.git**

5. Отправляем (загружаем) заданную главную ветку на удаленный сервер GitHub (первоначально необходимо авторизоваться в GitHub):
- **git push -u origin "branch name"**

6. В случае внесения изменения в рабочий файл на локальном репозитории (в программе Git на ПК), для актуализации данного файла на сервере GitHub необходимо выпрлнить команду:
- **git push**

PS. *Для изменения файла на удаленном репозитории на сайте GitHub - edit file - ввести коммит для сохранения.*

## Для "подтягивания" (скачивания) изменений актуализированного файла на удаленном репозиториий в файл на локальном репозитории:
- **git pull**
____

## **Для того, чтобы иметь возможность вносить изменения в файл, находящийся в чужом удаленном репозитории, а потом запросить "слияния" файлов необходимо:**

1. На необходимом (чужом) репозитории нажать кнопку **"Fork"**;

2. После создания "Форкнутого" репозитория на своей учетной записи удаленного репозитория, для внесения изменений в файлы, необходимо клонировать его на локальный репозиторий:
- в программе VSC кнопка "проводник" - "клонировать репозиторий" - выбрать необходимый из выпадающего списка вверху (первый раз необходима авторизация) 
- или командой **git clone "вставить адрес репозиория с сайта GitHub"**;

3. После изменений в файле сохраняем его "add", "commit". И отправляем на удаленный репозиторий командой **git push**.

4. Чтобы сделать запрос на слияние измененного файла с оригинальным репозиторием: кнопка "Pull requests" - "New pull request". В окне внизу, при необходимости, можно указать какие изменения были внесены и оставить сообщение создателю первоначального (оригинального) файла.
